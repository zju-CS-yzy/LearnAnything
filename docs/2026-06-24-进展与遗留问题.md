# LearnAnything 项目 — 2026-06-24 进展与遗留问题

## 今日进展 ✅

### 1. 向量数据库替换（核心修复）
- **问题**：ChromaDB 在 PyInstaller 打包后启动时崩溃（`chromadb.api.rust` 访问冲突）
- **解决**：完全移除 ChromaDB，改用 **SQLite + numpy** 实现自定义 `VectorStore`
  - 存储路径：`knowledge_base/vector_db/{collection_name}.db`
  - 使用 SQLite 存储文档和向量，numpy 进行 cosine similarity 计算
  - 兼容 BM25 缓存：`knowledge_base/cache/bm25_{collection}.pkl`

### 2. Embedding 服务迁移
- **问题**：本地 `sentence-transformers`（onnxruntime）在 PyInstaller 中启动失败
- **解决**：改用 **ZhipuAI `embedding-3` API**（2048 维）
  - 新增 `ApiEmbeddingClient` 类，支持 API 调用 + `HashEmbedding` 降级策略
  - 新增 `config/api_keys.ini` 管理 API 密钥（`zhipu_api_key`、`deepseek_api_key`）
  - 重建知识库：`ai_llm` 学科，279 篇文档，693 个 chunk

### 3. Reranker 降级策略
- **问题**：本地 `CrossEncoder` 在 PyInstaller 中无法加载
- **解决**：`RerankerFactory` 在 PyInstaller 模式下自动跳过 `CrossEncoder`，改用 `CosineReranker`
- 修复 `CosineReranker` 空列表处理 bug

### 4. LLM 回答生成
- `TutorAgent` 已实现调用 `LLMClient.chat()` 生成自然语言回答
- 缓存机制同时缓存 `chunks` 和 `answer`
- 后端 `/api/ask` 返回格式：`{answer: string, sources: [...], ...}`

### 5. PyInstaller 打包成功
- 移除 `chromadb`、`torch`、`sentence-transformers` 等 hiddenimports/excludes
- 打包后 exe 可正常运行，无启动崩溃

## 遗留问题 🔧

### 1. 前端显示仍为 JSON 格式（最优先）
- **现象**：用户点击"发送请求"后，后端返回 `answer`（长度 1617），但前端显示的是 JSON 序列化后的文本，而非自然语言回答
- **分析**：`apiAsk()` 函数在 `web/index.html` 中已被修改为直接设置 `el.textContent = data.answer`，但用户测试时仍看到 JSON
- **可能原因**：
  - PyInstaller 缓存问题：`build/app/web/index.html` 在删除 `build/` 后仍使用旧版本
  - 或 `web/index.html` 的修改未正确保存到打包目录 `_internal/web/index.html`
- **验证**：`dist/LearnAnything/_internal/web/index.html` 开头应为 `js-test` 元素，但最新打包后仍无该元素
- **解决方向**：
  - 方案 A：在 `app.spec` 的 `COLLECT` 之后添加手动复制 `web/index.html` 的步骤
  - 方案 B：使用 `simple.html` 作为最简前端替代方案

### 2. `web/index.html` 文件被意外覆盖
- 尝试用 `write` 工具修复文件时误写为 `0`，导致文件内容丢失
- 已用 `web/test_frontend.html` 恢复，但需确认内容完整
- **已修复**：`web/index.html` 已从 `test_frontend.html` 恢复，并重新应用 `apiAsk` 修改

### 3. API 返回内容编码问题（待确认）
- Python 控制台测试显示 `answer` 内容包含乱码（`\ufffd` 替换字符）
- 可能是 Windows 控制台 GBK 编码问题，而非实际数据损坏
- 需要验证：在浏览器中直接访问 `http://127.0.0.1:5001/api/ask` 返回的 JSON 是否正确

### 4. 前端调试信息不可见
- 原 `js-test` 元素使用 `position: fixed` 在 Qt WebEngine 中不显示
- 已改为页面顶部普通 `div`，但最新打包未包含此修改（缓存问题）
