## 附加优化：Markdown 渲染支持

当前 `apiAsk()` 使用 `textContent = data.answer`，这会原样显示文本，**不会渲染 Markdown**（如 `**粗体**`、`- 列表`、```代码块``` 等）。

### 方案：引入 marked.js 渲染 Markdown

在 `web/index.html` 的 `<head>` 中添加：

```html
<script src="https://cdn.jsdelivr.net/npm/marked/marked.min.js"></script>
```

修改 `apiAsk()` 函数：

```javascript
if (data.answer) {
    // 使用 marked.js 渲染 Markdown
    el.innerHTML = marked.parse(data.answer);
} else {
    el.textContent = JSON.stringify(data, null, 2);
}
```

并添加 Markdown 样式：

```css
.result {
    /* 现有样式保持不变 */
}
.result h1, .result h2, .result h3 {
    margin: 12px 0 8px;
    color: #2c3e50;
}
.result p {
    margin: 8px 0;
    line-height: 1.6;
}
.result ul, .result ol {
    margin: 8px 0;
    padding-left: 20px;
}
.result code {
    background: #f4f4f4;
    padding: 2px 6px;
    border-radius: 3px;
    font-family: 'Consolas', monospace;
    font-size: 13px;
}
.result pre {
    background: #f8f9fa;
    padding: 12px;
    border-radius: 8px;
    overflow-x: auto;
    margin: 10px 0;
}
.result pre code {
    background: none;
    padding: 0;
}
.result blockquote {
    border-left: 4px solid #667eea;
    margin: 10px 0;
    padding-left: 12px;
    color: #555;
}
```

### 离线支持（避免 CDN 依赖）

如果应用需要离线运行，将 `marked.min.js` 下载到 `web/lib/marked.min.js`：

```powershell
cd D:\MyCS\AI\Project\LearnAnything
mkdir -p web/lib
Invoke-WebRequest -Uri "https://cdn.jsdelivr.net/npm/marked/marked.min.js" -OutFile "web/lib/marked.min.js"
```

然后在 HTML 中引用：

```html
<script src="lib/marked.min.js"></script>
```

## 完整修复脚本

创建 `fix_and_rebuild.ps1`：

```powershell
# LearnAnything 前端修复脚本
$projectRoot = "D:\MyCS\AI\Project\LearnAnything"
Set-Location $projectRoot

Write-Host "=== Step 1: 清理 PyInstaller 缓存 ===" -ForegroundColor Cyan
if (Test-Path "build") {
    Remove-Item -Recurse -Force "build"
    Write-Host "已删除 build/ 目录" -ForegroundColor Green
}
if (Test-Path "dist") {
    Remove-Item -Recurse -Force "dist"
    Write-Host "已删除 dist/ 目录" -ForegroundColor Green
}

Write-Host "`n=== Step 2: 下载 marked.js（离线 Markdown 渲染）===" -ForegroundColor Cyan
if (-not (Test-Path "web/lib")) {
    New-Item -ItemType Directory -Path "web/lib" -Force
}
if (-not (Test-Path "web/lib/marked.min.js")) {
    try {
        Invoke-WebRequest -Uri "https://cdn.jsdelivr.net/npm/marked/marked.min.js" -OutFile "web/lib/marked.min.js" -TimeoutSec 10
        Write-Host "已下载 marked.min.js" -ForegroundColor Green
    } catch {
        Write-Host "下载失败，将使用 CDN 版本" -ForegroundColor Yellow
    }
}

Write-Host "`n=== Step 3: 重新打包 ===" -ForegroundColor Cyan
pyinstaller app.spec --noconfirm

Write-Host "`n=== Step 4: 验证 ===" -ForegroundColor Cyan
$packagedFile = "dist/LearnAnything/_internal/web/index.html"
if (Test-Path $packagedFile) {
    $size = (Get-Item $packagedFile).Length
    if ($size -gt 1000) {
        Write-Host "打包成功！文件大小: $size 字节" -ForegroundColor Green
    } else {
        Write-Host "警告：打包后文件可能损坏（大小: $size 字节）" -ForegroundColor Red
    }
} else {
    Write-Host "错误：打包后文件不存在" -ForegroundColor Red
}

Write-Host "`n=== 完成 ===" -ForegroundColor Cyan
Write-Host "运行: dist\LearnAnything\LearnAnything.exe"
```

## 后续建议

1. **每次修改前端后**：务必执行 `rmdir /s /q build` 再重新打包
2. **CI/CD 流程**：在打包脚本中自动添加清理步骤
