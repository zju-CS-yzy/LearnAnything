### LA-035-P21: 公式图片识别为 LaTeX

**问题修正**: MinerU 从 PDF 提取的公式实际是图片，不是文本 LaTeX。当前流程中这些公式图片被当作普通图片用 VLM 描述，导致 SemanticExtractor 无法获取精确的公式语义。

**修改文件**:
1. `core/image_concept_extractor.py`
   - 增加公式图片检测逻辑（基于宽高比：宽 ≥ 1.5×高、高度 ≤ 120px、宽度 ≥ 50px）
   - `_describe_image_with_context()` 自动判断图片类型，公式图片调用 VLM `task="formula"`
   - `_is_formula_image()` 基于 PIL 宽高比判断
   - `_recognize_formula()` VLM 公式识别 + LaTeX 清理（去除 markdown 代码块标记和 $$ 包裹）
   - `_create_pseudo_chunk()` 支持公式类型：text 为 LaTeX、media_refs 同时包含 image + formula

2. `core/mineru_client.py`
   - "Markdown 中无图片引用"的 image_pseudo chunks 也使用智能分析
   - 公式图片的 pseudo_chunk text 为 LaTeX 而非图片描述

3. `core/markdown_chunker.py`
   - `_extract_formulas()` 从文本中提取 `$...$` / `$$...$$` LaTeX 为 media_refs
   - `_build_media_refs()` 合并图片引用和公式为统一 media_refs
   - HeadingChunk 和 ParagraphChunk 的 metadata 均包含 `media_refs`

4. `docs/leftover-problem.md`
   - 新增 P21（修正方向）、P23（删除学科前端方法）、P24（media_ref 数据链断裂）

**测试脚本**:
- `scripts/test_formula_extraction.py` — 验证 markdown_chunker 的文本 LaTeX 提取
- `scripts/test_formula_detection.py` — 验证 image_concept_extractor 的公式图片宽高比检测

**遗留问题**:
- 端到端验证未完成：需要重新构建含公式的学科（如线性代数课件）验证全流程

- **前端修复**: `imageUrl`/`thumbnailUrl`/`getImageUrl` 兼容绝对路径；`renderFormulaContent` 将 `//` 替换为 `\`
