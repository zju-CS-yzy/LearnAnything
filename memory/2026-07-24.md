## 二、LA-056 悬空虚拟节点排查

### 诊断脚本
- `scripts/investigate_floating_v3.py` — 全量虚拟节点出度统计
- `scripts/investigate_child_identity.py` — 从 ID 推断 child 身份
- `scripts/trace_missing_edges.py` — 精确追踪缺失边

### 发现
- 89 个虚拟节点中，20 个悬空（入度=1，出度=0）
- 悬空虚拟节点的 child 都存在于数据库中
- 但 virtual→child 的边未被写入

### 根因
1. **类型 A（~50%）**: CycleDetector 过滤了 virtual→child 的边（形成环）
2. **类型 B（~50%）**: child 在其他树中，副本未生成，virtual→child 边指向外部节点

### 修复
1. `_cleanup_orphaned_virtual_nodes` 返回 `(cleaned_edges, orphaned_ids)`
2. 新增 `_delete_virtual_nodes` 从数据库删除 orphaned 节点
3. 脚本 `scripts/cleanup_floating_virtual.py` 清理已有数据

### 结果
- 89 → 69 虚拟节点
- 0 个悬空
- ✅ 已解决

## 三、LA-058 小树布局修复

### 根因
- 小树（≤5节点）用简单网格: `col=i%sCols, row=floor(i/sCols)`
- 未考虑父子关系，节点按索引排列成 L 形
- 间距 sGapY=140 小于节点实际高度（198px）

### 修复 v1（间距）
- 间距改为动态计算: `maxW+30, maxH+30`
- 结果: 节点不重叠，但仍呈网格状

### 修复 v2（树形布局）
- 改为 BFS 层次布局:
  1. 找根节点（入度=0）
  2. BFS 计算 level
  3. 水平: level 越大越靠右
  4. 垂直: 同层均匀分布

### 结果
- 小树形成清晰的从左到右树形结构
- ✅ 已解决

## 四、LA-057 双击布局失效修复

### 根因
- `expandNode()` 在概念视图下调用 `runTreeLayout()`（文档树布局）
- `runTreeLayout` 依赖 BELONGS_TO/ADJACENT_TO 等结构边
- 概念视图中没有这些边，导致布局崩溃

### 修复
`GraphView.vue`:
```javascript
if (viewMode.value === 'document') {
  runTreeLayout(cy)
} else {
  runConceptLayout(cy)
}
```

### 结果
- 双击后布局正常
- ✅ 已解决
