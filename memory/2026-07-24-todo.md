## 任务1: LA-044-#2 TutorAgent 个性化 Prompt

### 现状
- `TutorAgent.__init__` 已接受 `user_theta: float` 参数
- `_generate_answer()` 中系统提示词是硬编码，未使用 theta
- `AskRequest` 模型中没有 `user_theta` 字段

### 实施步骤
1. **定义 theta 区间和个性化指示**
   ```
   初学者 (0.0-0.3): 详细解释每个概念，多用类比，避免跳步，先解释"是什么"再解释"为什么"
   中级 (0.3-0.7): 适度深入，解释关键步骤，允许一定跳步，平衡直觉和形式化
   高级 (0.7-1.0): 简洁直接，聚焦核心原理，假设基础知识已知，深入技术细节
   ```

2. **修改 TutorAgent._generate_answer()**
   - 在系统提示词中添加个性化段落
   - 根据 user_theta 选择对应的指示

3. **修改 AskRequest 模型**
   - 添加 `user_theta: Optional[float] = Field(None, ge=0, le=1)`

4. **修改 Coordinator（如需要）**
   - 确保 user_theta 能传递到 TutorAgent

5. **测试**
   - 用相同的 question，分别传入 theta=0.1/0.5/0.9
   - 对比三个回答的深度差异

## 任务2: LA-044-#3 UserStateStore 双向同步

### 现状
- `UserStateStore` 已有完整的 SQLite CRUD
- 但对话结束后没有自动保存 theta

### 实施步骤
1. **新增 `/api/user-state` API**
   - GET: 获取当前用户的 theta/薄弱点
   - POST: 更新用户的 theta/薄弱点

2. **对话结束时自动保存**
   - 在 `/api/ask` 响应后，将更新后的 theta 写入 UserStateStore
   - 薄弱点从对话记录中提取（答错的知识点）

3. **测试**
   - 调用 POST /api/user-state 保存状态
   - 调用 GET /api/user-state 读取状态
   - 验证持久化
