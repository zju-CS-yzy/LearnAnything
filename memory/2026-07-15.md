## 关键发现

### KùzuDB 属性名差异
| 表名 | 主键字段 | 说明 |
|------|---------|------|
| Chunk | `chunk_id` | 非 `id` |
| ExtractedConcept | `extracted_id` | 非 `id` |
| CanonicalConcept | `canonical_id` | 非 `id` |

之前写查询脚本时用了 `id` 导致 `Binder exception: Cannot find property id for n`，浪费了不少时间排查。

### 数据库锁定问题
KùzuDB 在运行时（后端服务占用）无法被另一个进程连接。用户需要关闭后端服务才能查询。未来设计 debug 脚本时应考虑通过 API 而非直接连数据库，或者使用 read-only 模式。

### 学科数据库命名
- 实际使用的数据库文件名包含非 ASCII 字符（编码问题），导致 Python 脚本中 `os.listdir()` 返回乱码
- 使用 `glob.glob()` 替代可规避此问题
- 真正的大数据库是乱码名文件（62MB），而非 `rag_v1_graph`（也是乱码名但显示为 24KB）

### 数据验证脚本
创建了多个 debug 脚本：`scripts/debug_qkv_by_id.py`, `scripts/debug_canonical.py`, `scripts/debug_rag_db.py`, `scripts/debug_actual_db.py`, `scripts/debug_transformer_qkv.py`, `scripts/debug_qkv_edges.py` 等。可以在后续排查中复用或整合为一个通用诊断工具。
