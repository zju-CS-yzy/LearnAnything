### 4. 树布局交错问题修复（LA-026）
- **最终提交**: `3ffd230`
- **根因**: 51 节点分量含 8 个根节点，但整分量当作 1 棵树跑 dagre，导致视觉交错
- **修复历程**（多轮迭代）:
  | 提交 | 方案 | 结果 |
  |:---|:---|:---:|
  | `704330b` | 重建前端 v2.0 | ✅ 基础 |
  | `deeec7d` | 移除 cy.fit() + 收紧 dagre 参数 | ✅ 改善 |
  | `f0953a7` | 隐藏孤立节点 + 收紧 treeGap | ✅ 改善 |
  | `5cf1756` | 2D grid 布局 | ❌ 回退 |
  | `50f5ed6` | 全局 dagre TB | ❌ 回退 |
  | `6b13ba1` | 正确分量检测 + per-component dagre LR + 2D grid | ❌ 回退 |
  | `60ca22a` | vertical stack（每棵树独立区域）| ⚠️ 部分有效 |
  | `2f773f3` | 修复 container undefined | ✅ 修复 |
  | `3b5cf71` | debug bbox logging | ❌ 诊断 |
  | `cf18c20` | manual bbox + tighter dagre + larger treeGap | ⚠️ 过度调整 |
  | `0ac744c` | 移除过早的节点隐藏 | ✅ 关键修复 |
  | `3ffd230` | 重置位置后再跑 dagre | ✅ **最终方案** |
  | `d8b4f61` | 副本处理后按根拆分为独立子树 | ✅ 最终合并 |
- **最终方案**:
  1. 正确识别连通分量（无向图 BFS）
  2. 每个连通分量内：多入度节点创建副本 → 按根拆分为独立子树
  3. 每棵子树独立跑 dagre LR（重置位置避免累积）
  4. 多棵树纵向堆叠（从大到小排列，treeGap=100）
- **效果**: 恢复到 7月2日版本效果 ✅
